# Veritas vs ELIS 对比分析（2026-06-25 更新）

> 本次更新基于代码实际状态，修正了上一版中已过时的结论。

---

## 1. 项目定位

| 维度 | ELIS | Veritas |
|------|------|---------|
| **定位** | 通用科研图像诚信平台；多租户 SaaS | 干实验论文风控工具；内部 CLI-first |
| **受众** | 全球科研社区，开放注册 | 实验室导师和學生，内部使用 |
| **许可证** | AGPL-3.0 开源 | 不开源（内部工具） |
| **核心隐喻** | "让科研诚信工具民主化" | "帮导师在投稿前发现学生数据问题" |

**结论**：两个项目定位根本不同，不是简单的超集或替代关系。ELIS 是通用图像取证平台，Veritas 是垂直领域风控工具。

## 4. Veritas 是否是 ELIS 的超集？

**分层回答：不是简单的是/否。**

### 核心视觉取证能力：✅ 基本是超集

| ELIS 能力 | Veritas 状态 |
|-----------|-------------|
| Panel Extraction (YOLOv5) | ✅ 已覆盖 |
| Copy-Move Keypoint (RootSIFT+MAGSAC++) | ✅ 已覆盖 |
| Copy-Move Dense (SILA) | ✅ 已覆盖（AGENT_SELECTABLE） |
| TruFor forgery detection | ✅ 已覆盖（需 GPU） |
| Provenance 递归 BFS + MST | ✅ 已覆盖（ELIS adapter） |
| CBIR 跨论文检索 | ⚠️ 基础就绪，未接入自动审计主链路 |

**结论**：ELIS 的所有核心视觉取证算法，Veritas 都通过 adapter 或自研方式覆盖。唯一的缺口是 CBIR 的 SSCD embedding 未接入自动审计流程（Web API 可用但需手动触发）。

### 编排与报告能力：🟢 Veritas 远超 ELIS

ELIS 没有的能力：Source Data 全系列检测、PaperFraud 规则匹配、Agent 编排（29 tool_id + 3 role runners）、结构化 HTML 报告（3812 行）、Numeric Forensics（Paperconan GRIM/GRIMMER）、Claim Mapping、Overlap/Reuse 自研检测。

**结论**：这不是"超集"的问题——Veritas 在 ELIS 完全没有触及的维度建立了完整能力栈。

### 平台功能：ELIS 更完整，但 Veritas 刻意不做

用户管理、多租户、存储配额、Annotation 系统、i18n、PDF Viewer、Client-side Screening Tools、Watermark Removal——这些是 ELIS 作为开源 SaaS 平台的功能，Veritas 作为内部工具**不需要**。

**结论**：不是"Veritas 缺少这些功能"，而是"这些功能不在 Veritas 的产品边界内"。

## 6. UI/UX 差异与建议

### 6.1 设计哲学对比

| 维度 | ELIS | Veritas |
|------|------|---------|
| **视觉隐喻** | Glassmorphism / 科技感（indigo 渐变、backdrop-blur、圆角卡片） | Dossier / 案卷感（温暖纸张色调、墨水排版、IBM Plex Sans） |
| **导航模式** | 工具驱动（每个分析工具是一个页面） | 工作流驱动（Case Flow → Investigation Flow） |
| **用户角色** | 主动调查者（用户选择图像、触发工具、手动标注） | 审阅者（Agent 自动发现 finding，用户审阅和跟进） |
| **状态持久化** | sessionStorage | URL query params（支持深度链接） |
| **暗色模式** | 完整 toggle（dark: variants） | 固定 dark-on-paper 设计（无 toggle） |
| **数据密度** | 高（Analysis Dashboard 2500+ 行，List/Grid/Split 三视图） | 中（Mission Control 聚焦进度监控） |

### 6.2 Veritas 独有优势（不应放弃）

- **Dossier/案卷隐喻**：纸张色调 + 墨水排版 + 网格背景，强化" forensic report"的严肃感。这是有意的 UX 设计，不是审美选择
- **RiskTrafficLight**：4 级风险信号灯，一目了然
- **ProgressTracker**：步骤时间线 + emoji 图标，审计进度可视化
- **FollowUpDisplay**：一键复制待跟进问题
- **Benign explanations + manual review questions**：每个 finding 附带"可能的良性解释"和"需要人工确认的问题"——这是 ELIS 完全没有的模式，体现"只讲事实，不讲观点"的产品哲学
- **URL 深度链接**：`?page=findings&case=xxx&run=yyy`，支持分享和书签

### 6.3 可从 ELIS 借鉴的设计模式

| 模式 | ELIS 出处 | Veritas 改进建议 | 优先级 |
|------|-----------|------------------|--------|
| **Split-view master-detail** | `FlaggedImagesPage`（左列表 + 右详情面板） | `EvidenceReviewPage` 可改为左右分栏：左侧 finding 列表，右侧证据详情（panel 对比图、overlay mask、metrics） | P2 |
| **Tab-based detail panel** | `FlaggedImageDetailPanel`（Details / Annotations / Analysis / Related 标签页） | Finding 卡片可加标签页：Summary / Visual Evidence / Source Data / Related Claims | P3 |
| **Inline zoom controls** | `AnalysisDashboardPage`（Ctrl+滚轮缩放、zoom in/out/reset 按钮） | 证据图片（TruFor heatmap、copy-move matches）增加缩放交互 | P3 |
| **Batch selection toolbar** | `SelectionToolbar`（Shift-click 范围选择 + 浮动操作栏） | 多 panel 选择操作（批量 copy-move 分析、批量标记） | P3 |
| **Analysis Dashboard 式汇总** | List/Grid/Split 三视图 + 类型/状态/日期过滤 | 历史审计汇总视图：跨 case 查看所有 run 的 finding 分布、风险趋势 | P2 |
| **Wizard/stepped flow** | `ProvenancePage`（StepIndicator 三步向导） | 针对性分析工作流（选 panel → 选工具 → 看结果） | P3 |

### 6.4 不建议照搬的 ELIS 设计

- **Glassmorphism / 渐变风格**：与 Veritas dossier 隐喻冲突
- **工具驱动的侧边栏导航**：Veritas 工作流驱动更适合审计场景
- **手动 annotation 系统**：Veritas 的 Agent 自动生成 finding，不需要手动标注
- **Notifications SSE dropdown**：Veritas 的 Mission Control 已有实时进度监控

### 6.5 当前 Veritas 前端最大改进机会

1. **EvidenceReviewPage 改为 Split-view**（P2）：当前是垂直堆叠布局，finding 列表和证据详情上下排列，需要大量滚动。改为左右分栏可大幅提升审查效率。

2. **Finding 卡片增加可视化深度**（P2）：当前 finding 是纯文本摘要 + risk pill。可增加内联的 panel 对比缩略图、TruFor heatmap 预览，让用户不进入 EvidenceReview 也能快速判断。

3. **历史审计汇总视图**（P2）：CasesPage 当前按风险等级分组显示 case，但缺少跨 case 的 finding 统计、趋势图、高频问题类型分布。

## 7. 结论与优先级建议

### Veritas 是否是 ELIS 的超集？

**在核心视觉取证算法层面：是。** ELIS 的所有取证算法（panel extraction、copy-move keypoint/dense、TruFor、provenance 递归 BFS、CBIR）都已通过 adapter 或自研方式覆盖。

**在编排和报告层面：远超。** Source Data 检测、Agent 编排、PaperFraud 规则、结构化报告、Claim Mapping 是 ELIS 完全没有的能力维度。

**在平台功能层面：刻意不做。** 用户管理、annotation、i18n 等是 ELIS 作为 SaaS 的功能，不是 Veritas 的产品边界。

### 开发优先级

| 优先级 | 项目 | 预期工作量 | 产出 |
|--------|------|-----------|------|
| **P2** | CBIR SSCD 接入主链路（替代 HSV 直方图） | 1 周 | 补齐跨论文相似图检索 |
| **P2** | EvidenceReviewPage 改为 Split-view | 3 天 | 审查效率提升 |
| **P2** | 历史审计汇总视图 | 3 天 | 跨 case 分析能力 |
| **P3** | Finding 卡片增加内联可视化 | 2 天 | 快速判断效率 |
| **P3** | 证据图片 zoom controls | 1 天 | 交互体验提升 |
| **不做** | 用户管理 / annotation / i18n / screening tools / watermark removal | — | 不符合产品定位 |
