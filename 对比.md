## ELIS 架构真相

**重要纠正**：我深入分析了 ELIS 代码库，发现 **ELIS 根本没有使用 agents**。

**ELIS 实际架构**：
```
FastAPI (API 层)
    ↓
Celery Workers (异步任务队列)
    ↓
MongoDB + Redis (数据存储)
    ↓
Docker Containers (封装各个工具)
```

**关键发现**：
- ELIS 使用 Celery 做异步任务调度，不是 agents
- 没有 LLM-based 的决策层
- 所有工具调用都是预定义的固定流程
- 用户通过 Web UI 手动触发各个分析步骤

## Veritas vs ELIS 对比

| 能力           | ELIS              | Veritas                                          |

| **Agent 驱动** | ❌ 无             | ✅ 完整的 agent 调查流程                         |

**结论**：Veritas 在 agent 编排层面已经超越 ELIS。
