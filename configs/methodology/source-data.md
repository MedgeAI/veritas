# Source Data 静态审查方法论

Source Data 是 Veritas 静态审查的最高优先级证据层之一。

## 必查对象

- workbook、sheet、隐藏 sheet、公式单元格、外部链接、命名区域、注释。
- 合并单元格、多层表头、空值结构、分组标签、单位和统计说明。
- 原始值、派生值、归一化值、百分比、均值、SD、SEM、CI。

## 通用异常线索

- 整列重复、整行重复、长连续重复序列。
- 固定差、固定比、线性变换、重复小数部分。
- 末位 0/5 集中、过于整齐的小数、异常圆整。
- 公式派生列、反向构造的百分比/分母关系。
- 同一数据在不同 figure/panel/condition 中机械复用。

## 数字纹理和适用边界

### 小数一致性

优先作为弱到中等强度线索使用，必须和具体列、sheet、行范围绑定：

- 同一列或同一组独立测量中，小数位数异常一致，例如大量值都保留相同非约定精度。
- 小数部分重复，例如多个不同整数部分共享同一后两位、后三位或完整小数字符串。
- 小数各位数字分布明显不均，或相邻记录的小数部分出现异常自相关。
- 多组数据的小数纹理相同，但论文语义声称为独立实验、不同处理组或不同时间点。

使用时先排除：仪器固定精度、软件统一格式化、四舍五入、单位换算、百分比列、归一化列、导出模板、人工录入规范。

### 末位数字和 Benford

末位数字优先用于普通实验连续测量，Benford 必须先做适用性判断：

- 末位数字适用：体重、器官质量、荧光强度、细胞比例、浓度等连续测量，且样本量足够。
- 末位数字弱信号：样本很小、值经过强制圆整、只展示均值/SEM、或来自离散评分量表。
- Benford 适用：自然产生的数量值、样本量较大、跨多个数量级、没有强边界或人为编号规则。
- Benford 不适用：窄范围生物测量、比例/百分比、p 值、fold change、0-1 归一化值、ID/编号、几十个以内的小样本。

Benford 偏离不能单独升级为高风险；只有与重复值、固定关系、反向百分比、GRIM/GRIMMER 或图像复用等独立证据叠加时，才提高人工复核优先级。

### 趋势和模板化曲线

趋势/模板信号只在能回指 Source Data 行列或图表数值时进入 finding：

- 多个独立组呈现几乎相同的上升、下降、峰谷或平台形状。
- 一条曲线看起来由另一条曲线加减常数、乘除比例、平移或轻微扰动生成。
- 噪声过度平滑，或不同独立组的噪声纹理过于相似。
- 数据异常缺少自然离群点、批次差异或生物学变异。

优先作为人工复核任务呈现：要求核对原始测量记录、导出脚本、归一化流程、排序规则和是否存在共享 control / calibration curve。

## 误报排除顺序

1. 是否是编号列、样本 ID、lane 编号、坐标轴、时间点、浓度梯度。
2. 是否是设计矩阵、dummy variable、group label、contrast label。
3. 是否是公式派生、单位换算、归一化、背景扣除、百分比列。
4. 是否是合法 control 复用、同一数据多视角展示、技术重复。
5. 是否由合并单元格、多层表头、空值填充或解析器造成。

只有在排除上述路径后，才提升人工复核优先级。

## 输出要求

每个 Source Data finding 至少包含：

- workbook、sheet、row/column/cell 范围。
- 原始值样例。
- finding category。
- support rows / overlap rows / support rate。
- artifact likelihood。
- benign explanations。
- pressure-test result。
- manual review note。

claim-to-source-data 映射第一版允许到 sheet / figure 级，但必须标注尚未达到 panel 级强确认。
